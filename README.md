# 𓃥 白六滑動拼圖 (White 6 Sliding Puzzle)

一款基於 HTML5 Canvas 開發的極簡風「條狀滑動拼圖」網頁遊戲。玩家可以自訂上傳任意圖片，遊戲會自動進行等比裁切（Center Crop）並根據關卡難度將圖片隨機切分成數個水平或垂直的圖像條。玩家必須透過滑動行列來拼接、還原圖片。

本遊戲特別針對高關卡加入了極限挑戰的動態設限規則，並採用完全無縫（Seamless）的渲染模式，提供具有深度趣味性與高挑戰性的解謎體驗。

---

## 🎮 遊戲核心玩法

1. **上傳圖片**：點擊下方的「選擇或上傳新圖片」按鈕，挑選任何您喜歡的照片、生活照或插畫（預設為內建的幾何漸層背景）。
2. **滑動還原**：
   * 遊戲會隨機決定本關為**橫切（列滾動）**或**縱切（欄滾動）**。
   * 用手指或滑鼠按住畫布上的某一行（或某一列），往左右（或上下）拖曳滑動，即可循環滾動該條圖像。
   * 當所有切分圖像條皆完美對齊、恢復原本圖片的完整樣貌時，即算成功通關！
3. **無縫挑戰**：圖塊之間**完全沒有人工分隔線**。當圖塊完全對齊歸位時，圖像會完美無縫拼合，考驗玩家對畫面紋理與線條連續性的敏銳觀察力。

---

## 🏆 關卡與魔王設限機制

遊戲的切分難度會隨著關卡數（Level）線性遞增：
* **第 $N$ 關的切分規格**為：`主切分數 (Level + 1) × 副切分數 (Level + 2)`
  * *例如：第 1 關為 2 × 3 切分；第 2 關為 3 × 4 切分... 以此類推。*

為了兼顧熱身樂趣與核心玩家的極限競技需求，遊戲分為兩大階段：

### 🟢 1. 常規晉級安全區（第 1 ~ 20 關）
* **無時間壓力**：畫面上方會記錄本關耗時，但即使本關花費時間比上一關長，依然可以順利晉級。
* **沿用原圖**：過關後可自由選擇「下一關 (沿用原圖)」繼續遊玩，不需重選照片。

### 🔴 2. 地獄極限挑戰區（第 21 關起 💥）
一旦玩家成功踏入**第 21 關**，系統將自動啟動兩大嚴苛的設限條件：
1. **取消沿用原圖**：過關畫面中的「沿用原圖」按鈕將被強制隱藏，玩家**每過一關都必須點擊「換新圖片」**重新上傳，大幅提高畫面辨識與記憶的難度。
2. **時間退步限制 (Time Trial Deathmatch)**：
   * 狀態面板的「目標時間」會亮起紅色高壓提示，標準為**上一關的通關秒數**。
   * 如果玩家在當前關卡耗費的總秒數**高於上一關的紀錄**（即解題變慢了），在拼完的瞬間將會**觸發遊戲結束（Game Over）並被強制遣返回第 1 關**！
   * 只有維持相同速度或超越自我（花費更少時間）才能不斷往更高難度挺進。

## 🛠️ 技術特點與架構

* **純前端架構**：僅由單一一個 HTML5 檔案構成（包含 CSS 樣式與原生 JavaScript），無任何第三方套件或 Canvas 外部框架依賴，極速載入、安全且 100% 隱私（圖片皆在瀏覽器本地處理，絕不上傳伺服器）。
* **極致響應式設計**：
  * 使用 `aspect-ratio: 1 / 1` 與 CSS 變數維護正方形遊戲畫布，完美適應 iPhone、Android 行動裝置、平板與桌上型電腦。
  * 支援高解析度螢幕（利用 `window.devicePixelRatio` 放大 Canvas 畫布像素），確保上傳的高清圖片在視網膜螢幕上依然細緻銳利。
* **無縫渲染 (Seamless Rendering)**：移除格線並優化 Canvas 圖像邊緣的像素繪製，防止在條狀滑動與拼合時出現亞像素（Sub-pixel）白邊或裂縫。
* **雙重事件監聽**：同時完整綁定 `TouchEvents` 與 `MouseEvents`，動態計算滑動殘留量與格數四捨五入落點，確保滑動手感極度流暢。

## 🚀 快速開始

1. 將本專案的 `index.html` 檔案下載至您的電腦或手機。
2. 直接雙擊該 HTML 檔案，使用任意現代瀏覽器（Chrome, Safari, Edge, Firefox 等）打開。
3. 點擊「選擇或上傳新圖片」上傳您想挑戰的相片，即刻開啟您的「白六滑動拼圖」極限之旅！
